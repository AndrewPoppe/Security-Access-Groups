Einführung

FAQ

  1. Was ist es?

    Dies ist ein externes Modul, das REDCap-Administratoren die Erstellung und Verwaltung von Sicherheitszugriffsgruppen in einem REDCap-System ermöglicht.

  2. Was sind Sicherheitszugriffsgruppen?

    Security Access Groups (SAGs) werden verwendet, um einzuschränken, welche Benutzerrechte einem REDCap-Benutzer in einem Projekt gewährt werden können. SAGs definieren nicht die Rechte, die ein Benutzer in einem bestimmten Projekt haben wird; Vielmehr definieren sie den Satz zulässiger Rechte, die dem Benutzer gewährt werden können. SAGs werden auf Systemebene definiert und in jedem Projekt verwendet, in dem dieses Modul aktiviert ist.

  3. Wann wäre das sinnvoll?

    Wenn Ihre Institution beispielsweise eine Dokumentation der Schulung von Benutzern zum Exportieren von Daten benötigt, können Sie eine SAG erstellen, die es nicht zulässt, Benutzern, denen es zugewiesen wurde, das Benutzerrecht „Datenexporttool“ zu gewähren. Sobald ein Benutzer die Schulung abgeschlossen hat, kann er zu einer SAG verschoben werden, die die Gewährung des Benutzerrechts „Data Export Tool“ ermöglicht.

    Mit anderen Worten: Möglicherweise möchten Sie Ihren Benutzern unterschiedliche Schulungsebenen anbieten und bestimmte Benutzerrechte auf Projektebene nur Benutzern gewähren, die die entsprechende Schulung abgeschlossen haben. Obwohl dieses Modul keine Schulungen verwaltet oder verfolgt, kümmert es sich um alles andere.

  4. Liegt es nicht in der Verantwortung des Studienteams, die Benutzerrechte zu verwalten?

    Ja und nein. Als Systemadministratoren haben wir die Verantwortung sicherzustellen, dass die Benutzer unseres Systems für die von ihnen auszuführenden Aufgaben angemessen geschult werden. Viele Benutzer möchten keine stundenlange Schulung absolvieren, wenn sie nur Daten in ein Datenerfassungsgerät eingeben. Sie können diesen Benutzern eine kürzere, gezieltere Schulung geben und sie einer SAG zuweisen, die nur minimale Benutzerrechte zulässt.

Wie nutzen Sie das Modul?

REDCap-Administratoren erstellen zunächst SAGs über ein Menü im Control Center. Sobald eine SAG erstellt wurde, können ihr Benutzer auf einer anderen Registerkarte desselben Kontrollzentrumsmenüs zugewiesen werden. Wenn ein Benutzer einer SAG zugewiesen wird, können dem Benutzer nur Benutzerrechte gewährt werden, die von der SAG zugelassen werden.

🚩
 Das Modul muss in einem Projekt aktiviert sein, damit es die Benutzerrechte in diesem Projekt einschränkt. Es wird empfohlen, das Modul in allen Projekten zu aktivieren.

Wenn das Modul in einem Projekt aktiviert ist, prüft es, welche Rechte für einen Benutzer gemäß seiner SAG zulässig sind, bevor diesem Benutzer Benutzerrechte gewährt werden. Das Modul fängt alle Versuche ab, Benutzerrechte in einem Projekt festzulegen, einschließlich

  • Verwenden Sie die üblichen Methoden auf der Projektseite Benutzerrechte

  • Verwenden der Funktion „Benutzer, Rollen und Zuweisungen hochladen“ auf der Projektseite „Benutzerrechte“.

  • Verwendung der REDCap-API

Alle Versuche, Benutzerrechte festzulegen, werden blockiert, wenn dem Benutzer die Rechte gemäß seiner SAG nicht gewährt werden dürfen. Versuche, das Modul zu umgehen, werden erkannt und protokolliert.

💡

Das Modul ignoriert Benutzer, die derzeit im Projekt abgelaufen sind. Dies ermöglicht dem Benutzer, im Projekt zu bleiben, verhindert jedoch, dass er seine nicht konformen Benutzerrechte verwendet, bis diese nicht konformen Rechte entfernt werden. Der Benutzer kann dann nicht mehr abgelaufen sein.

Das Modul bietet eine praktische Möglichkeit, Benutzer in einem Projekt auf der Registerkarte „Projektstatus“ ablaufen zu lassen.

Registerkarte „Berichte“.

Abbildung 11. Registerkarte „Berichte“.

Diese Registerkarte bietet eine einfache Möglichkeit, alle Benutzer im System anzuzeigen, die derzeit über Benutzerrechte verfügen, die nicht ihrer aktuellen SAG entsprechen. Dies kann auftreten, wenn das Modul zum ersten Mal in einem Projekt aktiviert wird oder wenn ein Benutzer einer neuen SAG zugewiesen wird.

Die Berichtsoptionen sind wie folgt:

Titel des Berichts Beschreibung

Benutzer mit nicht konformen Rechten (nicht abgelaufen)

Dieser Bericht listet alle Benutzer auf, die einer SAG zugeordnet sind, die es nicht zulässt, dass dem Benutzer alle Rechte gewährt werden, die er aktuell in einem Projekt hat. Dieser Bericht umfasst nur Benutzer, wenn sie in den Projekten derzeit nicht abgelaufen sind.

Benutzer mit nicht konformen Rechten (alle)

Dieser Bericht listet alle Benutzer auf, die einer SAG zugeordnet sind, die es nicht zulässt, dass dem Benutzer alle Rechte gewährt werden, die er aktuell in einem Projekt hat. Dieser Bericht umfasst alle Benutzer, unabhängig davon, ob sie derzeit in den Projekten abgelaufen sind.

Projekte mit nicht konformen Rechten (nicht abgelaufen)

Dieser Bericht listet alle Projekte auf, die mindestens einen Benutzer haben, der einer SAG zugeordnet ist, die es dem Benutzer nicht ermöglicht, alle Rechte zu gewähren, die er derzeit im Projekt hat. Dieser Bericht umfasst nur Benutzer, die über ein nicht abgelaufenes Benutzerkonto verfügen.

Projekte mit nicht konformen Rechten (alle)

Dieser Bericht listet alle Projekte auf, die mindestens einen Benutzer haben, der einer SAG zugeordnet ist, die es dem Benutzer nicht ermöglicht, alle Rechte zu gewähren, die er derzeit im Projekt hat. Dieser Bericht umfasst alle Benutzer, unabhängig davon, ob ihr Benutzerkonto abgelaufen ist.

Benutzer und Projekte mit nicht konformen Rechten (nicht abgelaufen)

In diesem Bericht werden alle Benutzer- und Projektkombinationen aufgeführt, bei denen der Benutzer einer SAG zugeordnet ist, die es nicht zulässt, dass dem Benutzer alle Rechte gewährt werden, die er derzeit im Projekt hat. Dieser Bericht umfasst nur Benutzer, deren Mitgliedschaft im Projekt derzeit noch nicht abgelaufen ist.

Benutzer und Projekte mit nicht konformen Rechten (alle)

In diesem Bericht werden alle Benutzer- und Projektkombinationen aufgeführt, bei denen der Benutzer einer SAG zugeordnet ist, die es nicht zulässt, dass dem Benutzer alle Rechte gewährt werden, die er derzeit im Projekt hat. Dieser Bericht umfasst alle Benutzer, unabhängig davon, ob sie derzeit im Projekt abgelaufen sind.

Projektseite

Registerkarte „Projektstatus“.

Das Modul fügt eine Seite hinzu, die den Status aller Benutzer im Projekt anzeigt. Der Status jedes Benutzers wird durch die SAG des Benutzers und die Rechte bestimmt, die er im Projekt hat. Die Farbe der Zeile zeigt an, ob der Benutzer:

  • Grün – entspricht der SAG

  • Rot – entspricht nicht der SAG

  • Grau – im Projekt abgelaufen

Sie können auch die Spalte „Nicht konforme Rechte“ überprüfen, um zu sehen, welche Rechte der Benutzer hat, die von seiner SAG nicht zugelassen werden.

💡
 Wenn Sie nur Benutzer sehen möchten, die fälschlicherweise über bestimmte Rechte verfügen, können Sie das Suchfeld verwenden. Wenn Sie beispielsweise nur an den Benutzerrechten und/oder Projektdesign- und Setup- Rechten interessiert sind, geben Sie "user rights" | "project design" ein Geben Sie im Suchfeld "user rights" | "project design" . Weitere Informationen finden Sie im Abschnitt „Suchen“ .

Protokollierung

Einer der Vorteile dieses Moduls ist die verbesserte Protokollierung. Das Modul erstellt detaillierte Protokolle in den projekteigenen Protokollen für alle Änderungen an Benutzerrechten, einschließlich

  • Wenn ein Benutzer mit benutzerdefinierten Rechten zu einem Projekt hinzugefügt wird

  • Wenn ein Benutzer einem Projekt in einer vorhandenen Benutzerrolle hinzugefügt wird

  • Wenn die Rechte eines Benutzers geändert werden

  • Wenn die Rechtedefinition einer Rolle geändert wird

  • Wenn ein Benutzer einer Benutzerrolle hinzugefügt/entfernt wird

  • Wenn Benutzer per CSV in ein Projekt importiert werden

  • Wenn die Rechte eines Benutzers per CSV-Import geändert werden

  • Wenn Rollen per CSV in ein Projekt importiert werden

  • Wenn ein Benutzer per CSV-Import einer Rolle zugewiesen wird

  • Wenn ein Benutzer über die API zu einem Projekt hinzugefügt wird

  • Wenn die Rechte eines Benutzers über die API geändert werden

  • Wenn Benutzerrollen über die API importiert/geändert werden

  • Wenn ein Benutzer über die API einer Rolle zugewiesen wird

Einschränkungen

Rechts-nach-links-Sprachen

Derzeit gibt es nur begrenzte Unterstützung für RTL-Sprachen. Das Modul zeigt RTL-Sprachen korrekt an, aber die Struktur/Formatierung von UI-Elementen ist weiterhin LTR. Es gibt Optionen für eine umfassendere Unterstützung von RTL, diese wird jedoch eine niedrige Priorität haben, es sei denn, wir hören von Gruppen, die diese Funktion benötigen.

Übersetzen des Moduls

Die mit dem Modul bereitgestellten Übersetzungen wurden mit automatischer Übersetzungssoftware erstellt und sind möglicherweise nicht korrekt. Wenn Sie eine Übersetzung korrigieren oder eine neue Übersetzung beisteuern möchten, befolgen Sie bitte die nachstehenden Anweisungen.

Erstellen einer neuen Übersetzung

Wenn Sie das Modul in eine neue Sprache übersetzen möchten, können Sie dies tun, indem Sie die folgenden Schritte ausführen:

  1. Suchen Sie das lang Verzeichnis im Quellcode des Moduls.

  2. Kopieren Sie die Datei English.ini und ändern Sie den Namen der Kopie in die Sprache, in die Sie übersetzen möchten. Benennen Sie die Datei mit dem englischen Namen der Sprache (in Großbuchstaben), gefolgt vom Endonym der Sprache (unter Verwendung der Glyphen dieser Sprache) in Klammern. Wenn Sie das Modul beispielsweise ins Japanische übersetzen möchten, würden Sie die Datei Japanese (日本語).ini nennen.

  3. Öffnen Sie die soeben erstellte Datei in einem Texteditor und übersetzen Sie den Text auf der rechten Seite des Gleichheitszeichens für jede Zeile. Wenn Sie beispielsweise den Text Introduction ins Japanische übersetzen möchten, ändern Sie die Zeile status_ui_3 = "Introduction" in status_ui_3 = "序章" .

  4. Wiederholen Sie den Vorgang für jede Zeile.

  5. Speichern Sie die Datei und laden Sie sie in den Ordner „lang“ des Quellcodes des Moduls hoch.

  6. Senden Sie eine Pull-Anfrage mit Ihren Änderungen an den Master-Zweig des Github-Repositorys .

Korrigieren einer vorhandenen Übersetzung

Wenn Sie eine vorhandene Übersetzung korrigieren möchten, können Sie dies mit den folgenden Schritten tun:

  1. Suchen Sie das lang Verzeichnis im Quellcode des Moduls.

  2. Öffnen Sie die Datei für die Sprache, die Sie korrigieren möchten, in einem Texteditor.

  3. Korrigieren Sie den Text auf der rechten Seite des Gleichheitszeichens für jede Zeile, die Sie ändern möchten.

  4. Speichern Sie die Datei und laden Sie sie in den Ordner „lang“ des Quellcodes des Moduls hoch.

  5. Senden Sie eine Pull-Anfrage mit Ihren Änderungen an den Master-Zweig des Github-Repositorys .

Suchen

Allgemein

Viele der Tabellen im Modul verfügen über ein Suchfeld, mit dem die Tabelle gefiltert werden kann. Das Suchfeld durchsucht alle Spalten in der Tabelle.

Wenn Sie beispielsweise alle Benutzer finden möchten, die derzeit der SAG zugeordnet sind, deren Bezeichnung das Wort 'Nothing' enthält, können Sie 'Nothing' in das Suchfeld eingeben und die Tabelle wird so gefiltert, dass nur Zeilen angezeigt werden, die das Wort „Nothing“ enthalten den Text 'Nothing' in einer beliebigen Spalte.

„AND“-Suchen

Standardmäßig handelt es sich bei dem Suchfeld um eine „UND“-Suche, was bedeutet, dass Ihr Suchbegriff in einzelne Wörter aufgeteilt wird und nur Ergebnisse angezeigt werden, die mit allen diesen Suchbegriffen übereinstimmen. Wenn Sie beispielsweise 'joe admin' in das Suchfeld eingeben, wird die Tabelle so gefiltert, dass nur Zeilen angezeigt werden, die SOWOHL 'joe' ALS AUCH 'admin' in einer Spalte enthalten. Es werden keine Zeilen angezeigt, die entweder 'joe' oder 'admin' enthalten, aber Zeilen, die z. B. 'admin' in Spalte 1 und 'joe' in Spalte 2 enthalten .

Abbildung 27. AND-Beispiel

„ODER“-Suchen

Wenn Sie alle Zeilen anzeigen möchten, die mit einem Wert ODER einem anderen Wert übereinstimmen, müssen Sie eine „ODER“-Suche verwenden.

Sie können das Zeichen „|“ verwenden. Operator, um nach mehreren Begriffen wie diesem entweder/oder zu suchen. Wenn Sie beispielsweise alle Benutzer mit dem Benutzernamen "alice" oder "bob" finden möchten, können Sie "alice | bob" in das Suchfeld eingeben und die Tabelle wird so gefiltert, dass nur Benutzer mit "alice" oder "bob" angezeigt werden "bob" in einer beliebigen Spalte.

💡
 einschließlich des '|' symbol hat den Nebeneffekt, dass die Suche zu einer Suche mit regulären Ausdrücken wird (siehe unten ).

Genaue Phrasensuche

Wenn Sie nach einer Phrase filtern möchten, können Sie die Phrase in doppelte Anführungszeichen setzen. Wenn Sie beispielsweise alle Zeilen mit der genauen Phrase 'joe admin' finden möchten, können Sie '"joe admin"' in das Suchfeld eingeben und die Tabelle wird so gefiltert, dass in jeder Spalte nur Zeilen mit 'joe admin' angezeigt werden.

Suche nach regulären Ausdrücken

Durch Einfügen eines '|' Geben Sie an einer beliebigen Stelle in Ihrem Suchbegriff ein Zeichen ein, wird Ihre Suche zu einer Suche mit regulären Ausdrücken . Dadurch können Sie komplexe Suchvorgänge zusammenstellen, die mit der Standardsuche nicht möglich sind.
