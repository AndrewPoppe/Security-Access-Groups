Introduction

FAQ

  1. Qu'est-ce que c'est?

    Il s'agit d'un module externe qui permet aux administrateurs REDCap de créer et de gérer des groupes d'accès de sécurité dans un système REDCap.

  2. Que sont les groupes d'accès sécurisé ?

    Les groupes d'accès de sécurité (SAG) sont utilisés pour restreindre les droits d'utilisateur qu'un utilisateur REDCap peut se voir accorder dans un projet. Les SAG ne définissent pas les droits qu'un utilisateur aura dans un projet donné ; ils définissent plutôt l'ensemble des droits autorisés que l'utilisateur peut se voir accorder. Les SAG sont définis au niveau du système et sont utilisés dans tout projet pour lequel ce module est activé.

  3. Quand cela serait-il utile ?

    Par exemple, si votre institution exige une documentation de formation pour que les utilisateurs exportent des données, vous pouvez créer un SAG qui n'autorise pas l'octroi du droit d'utilisateur "Outil d'exportation de données" aux utilisateurs qui lui sont attribués. Une fois qu'un utilisateur a terminé la formation, il peut être déplacé vers un SAG qui autorise l'octroi du droit d'utilisateur "Outil d'exportation de données".

    En d'autres termes, vous souhaiterez peut-être proposer différents niveaux de formation à vos utilisateurs et n'accorder des droits d'utilisateur spécifiques au niveau du projet qu'aux utilisateurs ayant suivi la formation appropriée. Bien que ce module n'administre ni ne suive la formation, il gère tout le reste.

  4. N'est-ce pas la responsabilité de l'équipe d'étude de gérer les droits des utilisateurs ?

    Oui et non. En tant qu'administrateurs système, nous avons la responsabilité de nous assurer que les utilisateurs de notre système sont correctement formés pour les tâches qu'ils effectueront. De nombreux utilisateurs ne souhaitent pas suivre une formation de plusieurs heures s'ils doivent uniquement saisir des données dans un instrument de collecte de données. Vous pouvez donner à ces utilisateurs une formation plus courte et ciblée et les affecter à un SAG qui accorde des droits d'utilisateur minimaux.

Comment utilisez-vous le module ?

Les administrateurs REDCap créent d'abord des SAG à l'aide d'un menu dans le centre de contrôle. Une fois qu'un SAG est créé, les utilisateurs peuvent lui être affectés dans un autre onglet du même menu du centre de contrôle. Lorsqu'un utilisateur est affecté à un SAG, l'utilisateur ne pourra se voir accorder que les droits d'utilisateur autorisés par le SAG.

🚩
 Le module doit être activé dans un projet pour qu'il restreigne les droits d'utilisateur dans ce projet. Il est recommandé d'activer le module dans tous les projets.

Lorsque le module est activé dans un projet, il vérifie quels droits sont autorisés pour un utilisateur en fonction de son SAG avant d'accorder des droits d'utilisateur à cet utilisateur. Le module intercepte toutes les tentatives de définition des droits des utilisateurs dans un projet, y compris

  • En utilisant les méthodes habituelles sur la page du projet User Rights

  • Utilisation de la fonctionnalité Charger des utilisateurs, des rôles et des affectations sur la page du projet Droits des utilisateurs

  • Utilisation de l'API REDCap

Toutes les tentatives de définition des droits d'utilisateur seront bloquées si l'utilisateur n'est pas autorisé à se voir accorder les droits en fonction de son SAG. Les tentatives de contournement du module sont détectées et consignées.

💡

Le module ignore les utilisateurs qui sont actuellement expirés dans le projet. Cela permet à l'utilisateur de rester dans le projet mais l'empêche d'utiliser ses droits d'utilisateur non conformes jusqu'à ce que ces droits non conformes soient supprimés. L'utilisateur peut alors être non expiré.

Le module offre un moyen pratique d'expirer les utilisateurs d'un projet sur l' onglet État du projet .

Onglet Rapports

Figure 11. Onglet Rapports

Cet onglet permet de voir facilement tous les utilisateurs du système qui ont actuellement des droits d'utilisateur qui ne sont pas conformes à leur SAG actuel. Cela peut se produire lorsque le module est activé pour la première fois dans un projet ou lorsqu'un utilisateur est affecté à un nouveau SAG.

Les options de rapport sont les suivantes :

Titre du rapport Description

Utilisateurs avec droits non conformes (non expirés)

Ce rapport répertorie tous les utilisateurs affectés à un SAG qui ne permet pas à l'utilisateur de se voir accorder tous les droits dont il dispose actuellement dans un projet. Ce rapport n'inclut que les utilisateurs s'ils n'ont pas encore expiré dans le(s) projet(s).

Utilisateurs avec droits non conformes (tous)

Ce rapport répertorie tous les utilisateurs affectés à un SAG qui ne permet pas à l'utilisateur de se voir accorder tous les droits dont il dispose actuellement dans un projet. Ce rapport inclut tous les utilisateurs, qu'ils aient expiré ou non dans le(s) projet(s).

Projets avec droits non conformes (non expirés)

Ce rapport répertorie tous les projets qui ont au moins un utilisateur affecté à un SAG qui ne permet pas à l'utilisateur de se voir accorder tous les droits dont il dispose actuellement dans le projet. Ce rapport inclut uniquement les utilisateurs dont le compte utilisateur n'a pas expiré.

Projets avec droits non conformes (tous)

Ce rapport répertorie tous les projets qui ont au moins un utilisateur affecté à un SAG qui ne permet pas à l'utilisateur de se voir accorder tous les droits dont il dispose actuellement dans le projet. Ce rapport inclut tous les utilisateurs, que leur compte d'utilisateur ait expiré ou non.

Utilisateurs et projets avec des droits non conformes (non expirés)

Ce rapport répertorie toutes les combinaisons d'utilisateurs et de projets dans lesquelles l'utilisateur est affecté à un SAG qui ne permet pas à l'utilisateur d'obtenir tous les droits dont il dispose actuellement dans le projet. Ce rapport inclut uniquement les utilisateurs qui n'ont pas encore expiré dans le projet.

Utilisateurs et projets avec des droits non conformes (tous)

Ce rapport répertorie toutes les combinaisons d'utilisateurs et de projets dans lesquelles l'utilisateur est affecté à un SAG qui ne permet pas à l'utilisateur d'obtenir tous les droits dont il dispose actuellement dans le projet. Ce rapport inclut tous les utilisateurs, qu'ils aient expiré ou non dans le projet.

Page du projet

Onglet État du projet

Le module ajoute une page qui affiche le statut de tous les utilisateurs du projet. Le statut de chaque utilisateur est déterminé par le SAG de l'utilisateur et les droits dont il dispose dans le projet. La couleur de la ligne indique si l'utilisateur est :

  • Vert - conforme à leur SAG

  • Rouge - non conforme à leur SAG

  • Gris - expiré dans le projet

Vous pouvez également consulter la colonne Droits non conformes pour voir quels droits l'utilisateur possède et qui ne sont pas autorisés par son SAG.

💡
 Si vous souhaitez voir uniquement les utilisateurs qui ont des droits particuliers de manière inappropriée, vous pouvez utiliser le champ de recherche. Par exemple, si vous n'êtes intéressé que par les droits d'utilisateur et/ou les droits de conception et de configuration du projet , tapez "user rights" | "project design" dans le champ de recherche. Voir la section Recherche pour plus d'informations.
Figure 14. Onglet État du projet
💡
Si des utilisateurs ne respectent pas leur SAG, vous pouvez utiliser l'un des boutons d'action pour envoyer un e-mail à l'utilisateur, aux titulaires de droits de l'utilisateur ou aux deux. Vous pouvez également faire expirer l'utilisateur du projet. Une alerte peut éventuellement être envoyée à l'utilisateur et/ou aux ayants droit de l'utilisateur lorsque l'utilisateur est expiré.

Enregistrement

L'un des avantages de l'utilisation de ce module est la journalisation améliorée qu'il fournit. Le module crée des journaux détaillés dans les propres journaux du projet pour toutes les modifications apportées aux droits d'utilisateur, y compris

  • Lorsqu'un utilisateur est ajouté à un projet avec des droits personnalisés

  • Lorsqu'un utilisateur est ajouté à un projet dans un rôle d'utilisateur existant

  • Lorsque les droits d'un utilisateur sont modifiés

  • Lorsque la définition des droits d'un rôle est modifiée

  • Lorsqu'un utilisateur est ajouté/supprimé d'un rôle d'utilisateur

  • Lorsque des utilisateurs sont importés dans un projet via CSV

  • Lorsque les droits d'un utilisateur sont modifiés via l'importation CSV

  • Lorsque les rôles sont importés dans un projet via CSV

  • Lorsqu'un utilisateur est affecté à un rôle via l'importation CSV

  • Lorsqu'un utilisateur est ajouté à un projet via l'API

  • Lorsque les droits d'un utilisateur sont modifiés via l'API

  • Lorsque les rôles d'utilisateur sont importés/modifiés via l'API

  • Lorsqu'un utilisateur est affecté à un rôle via l'API

Traduction (i18n / l10n)

Informations générales

Le module peut être traduit dans des langues autres que l'anglais via la fonction de sélection de langue de l'infrastructure de module externe. Pour définir la langue du module à l'échelle du système, accédez à Centre de contrôle > Modules externes > Gérer les modules externes et cliquez sur le bouton "Configurer" du module. Sélectionnez ensuite la langue que vous souhaitez utiliser dans la liste déroulante "Langue" et cliquez sur "Enregistrer".

Vous pouvez remplacer la langue du système au niveau du projet en visitant le gestionnaire de module de projet et en cliquant sur le bouton "Configurer" pour le module. Sélectionnez ensuite la langue que vous souhaitez utiliser dans la liste déroulante "Langue" et cliquez sur "Enregistrer".

🚩
 Certaines parties du module ne sont pas traduites par le module lui-même, mais reposent plutôt sur le système de traduction de langue intégré de REDCap. Ceux-ci incluent les noms des droits d'utilisateur et les descriptions associées.

Limites

Langues de droite à gauche

Actuellement, la prise en charge des langues RTL est limitée. Le module affichera correctement les langues RTL, mais la structure/le formatage des éléments de l'interface utilisateur sera toujours LTR. Il existe des options pour une prise en charge plus complète de RTL, mais cela ne sera pas prioritaire à moins que nous n'ayons des nouvelles de groupes qui ont besoin de cette fonctionnalité.

Traduire le module

Les traductions fournies avec le module ont été créées à l'aide d'un logiciel de traduction automatique et peuvent ne pas être exactes. Si vous souhaitez corriger une traduction ou apporter une nouvelle traduction, veuillez suivre les instructions ci-dessous.

Création d'une nouvelle traduction

Si vous souhaitez traduire le module dans une nouvelle langue, vous pouvez le faire en suivant ces étapes :

  1. Trouvez le répertoire lang dans le code source du module.

  2. Copiez le fichier English.ini et modifiez le nom de la copie dans la langue vers laquelle vous souhaitez traduire. Nommez le fichier avec le nom anglais de la langue (en majuscule) suivi de l'endonyme de la langue (en utilisant les glyphes de cette langue) entre parenthèses. Par exemple, si vous souhaitez traduire le module en japonais, vous devez nommer le fichier Japanese (日本語).ini .

  3. Ouvrez le fichier que vous venez de créer dans un éditeur de texte et traduisez le texte à droite du signe égal pour chaque ligne. Par exemple, si vous vouliez traduire le texte Introduction en japonais, vous remplaceriez la ligne status_ui_3 = "Introduction" par status_ui_3 = "序章" .

  4. Répétez le processus pour chaque ligne.

  5. Enregistrez le fichier et téléchargez-le dans le dossier "lang" du code source du module.

  6. Soumettez une demande d'extraction avec vos modifications à la branche principale du référentiel Github .

Corriger une traduction existante

Si vous souhaitez corriger une traduction existante, vous pouvez le faire en suivant ces étapes :

  1. Trouvez le répertoire lang dans le code source du module.

  2. Ouvrez le fichier correspondant à la langue que vous souhaitez corriger dans un éditeur de texte.

  3. Corrigez le texte à droite du signe égal pour chaque ligne que vous souhaitez modifier.

  4. Enregistrez le fichier et téléchargez-le dans le dossier "lang" du code source du module.

  5. Soumettez une demande d'extraction avec vos modifications à la branche principale du référentiel Github .

Divers

Recherche

Général

De nombreux tableaux du module comportent une zone de recherche qui peut être utilisée pour filtrer le tableau. La zone de recherche recherchera toutes les colonnes du tableau.

Par exemple, si vous souhaitez rechercher tous les utilisateurs actuellement affectés au SAG dont l'étiquette contient le mot 'Nothing' , vous pouvez taper 'Nothing' dans la zone de recherche et le tableau sera filtré pour afficher uniquement les lignes qui ont le texte 'Nothing' dans n'importe quelle colonne.

Figure 26. Exemple de recherche

Recherches "ET"

Par défaut, le champ de recherche sera une recherche "ET", ce qui signifie qu'il divisera votre terme de recherche en mots distincts et n'affichera que les résultats correspondant à tous ces mots de recherche. Par exemple, si vous tapez 'joe admin' dans la zone de recherche, le tableau sera filtré pour n'afficher que les lignes contenant à la fois 'joe' ET 'admin' dans n'importe quelle colonne. Il n'affichera pas les lignes qui ont soit 'joe' ou 'admin' mais il affichera les lignes qui ont par exemple, 'admin' dans la colonne 1 et 'joe' dans la colonne 2.

Figure 27. Exemple ET

Recherches "OU"

Si vous souhaitez afficher toutes les lignes qui correspondent à une valeur OU correspondent à une autre valeur, vous devez utiliser une recherche "OU".

Vous pouvez utiliser le "|" opérateur pour rechercher plusieurs termes comme celui-ci d'une manière ou/ou. Par exemple, si vous voulez trouver tous les utilisateurs avec le nom d'utilisateur "alice" ou "bob" , vous pouvez taper "alice | bob" dans la zone de recherche et le tableau sera filtré pour n'afficher que les utilisateurs avec "alice" ou "bob" . "bob" dans n'importe quelle colonne.

💡
 y compris le '|' a pour effet secondaire de faire de la recherche une recherche d'expression régulière (voir ci-dessous ).
Figure 28. Exemple OU

Recherches d'expressions exactes

Si vous souhaitez filtrer en fonction d'une phrase, vous pouvez placer votre phrase entre guillemets doubles. Par exemple, si vous souhaitez trouver toutes les lignes contenant l'expression exacte 'joe admin' vous pouvez saisir '"joe admin"' dans la zone de recherche et le tableau sera filtré pour n'afficher que les lignes contenant 'joe admin' dans n'importe quelle colonne.

Figure 29. Exemple de phrase exacte

Recherches d'expressions régulières

En incluant un '|' caractère n'importe où dans votre terme de recherche, votre recherche se transforme en une recherche d'expression régulière . Cela vous permet de composer des recherches complexes qui ne sont pas possibles avec la recherche par défaut.
