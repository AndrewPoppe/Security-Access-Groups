介绍

常问问题

  1. 它是什么？

    这是一个外部模块，允许 REDCap 管理员在 REDCap 系统中创建和管理安全访问组。

  2. 什么是安全访问组？

    安全访问组(SAG) 用于限制可以在项目中授予 REDCap 用户哪些用户权限。 SAG 不定义用户在给定项目中拥有的权限；相反，它们定义了用户能够被授予的一组允许的权限。 SAG 在系统级别定义，并在启用此模块的任何项目中使用。

  3. 什么时候会有用呢？

    例如，如果您的机构需要用户导出数据的培训文档，您可以创建一个 SAG，不允许将“数据导出工具”用户权限授予分配给该用户的用户。用户完成培训后，可以将他们转移到允许授予“数据导出工具”用户权限的 SAG。

    换句话说，您可能希望为用户提供不同级别的培训，并且仅向已完成适当培训的用户授予特定的项目级别用户权限。尽管该模块不管理或跟踪培训，但它确实处理其他所有事情。

  4. 管理用户权限不是研究组的责任吗？

    是和不是。作为系统管理员，我们有责任确保我们系统的用户针对他们将要做的工作接受充分的培训。如果许多用户仅在数据收集仪器中输入数据，则他们不想完成长达数小时的培训。您可以为这些用户提供更短的、有重点的培训，并将他们分配到允许最少用户权限的 SAG。

您如何使用该模块？

REDCap 管理员首先使用控制中心中的菜单创建 SAG。创建 SAG 后，可以在同一控制中心菜单的另一个选项卡中将用户分配给它。当用户被分配到 SAG 时，该用户将只能被授予该 SAG 允许的用户权限。

🚩
 必须在项目中启用该模块才能限制该项目中的用户权限。建议在所有项目中启用该模块。

当在项目中启用该模块时，它将根据用户的 SAG 检查用户允许哪些权限，然后再向该用户授予任何用户权限。该模块拦截项目中所有设置用户权限的尝试，包括

  • 使用用户权限项目页面上的常用方法

  • 使用用户权限项目页面上的上传用户、角色和分配功能

  • 使用 REDCap API

如果不允许用户根据其 SAG 被授予权限，则所有设置用户权限的尝试都将被阻止。检测并记录规避该模块的尝试。

💡

该模块会忽略项目中当前过期的用户。这允许用户保留在项目中，但阻止他们使用不合规的用户权限，直到这些不合规的权限被删除。然后用户可以取消过期。

该模块提供了一种在“项目状态”选项卡上使项目中的用户过期的便捷方法。

报告选项卡

图 11. 报告选项卡

此选项卡提供了一种简单的方法来查看系统中当前拥有不符合其当前 SAG 的用户权限的所有用户。当模块首次在项目中启用或用户被分配到新的 SAG 时，可能会发生这种情况。

报告选项如下：

报告标题 描述

具有不合规权利的用户（未过期）

此报告列出了分配给 SAG 的所有用户，该 SAG 不允许向用户授予他们当前在项目中拥有的所有权限。此报告仅包含项目中当前未过期的用户。

具有不合规权限的用户（全部）

此报告列出了分配给 SAG 的所有用户，该 SAG 不允许向用户授予他们当前在项目中拥有的所有权限。此报告包括所有用户，无论他们当前在项目中是否已过期。

具有不合规权利的项目（未过期）

此报告列出了至少有一名用户被分配到某个 SAG 的所有项目，而该 SAG 不允许向该用户授予他们当前在项目中拥有的所有权限。此报告仅包括拥有未过期用户帐户的用户。

权利不合规的项目（全部）

此报告列出了至少有一名用户被分配到某个 SAG 的所有项目，而该 SAG 不允许向该用户授予他们当前在项目中拥有的所有权限。此报告包括所有用户，无论其用户帐户是否已过期。

具有不合规权利的用户和项目（未过期）

此报告列出了每个用户和项目组合，其中用户被分配到不允许向用户授予其当前在项目中拥有的所有权限的 SAG。该报告仅包含项目中当前未过期的用户。

具有不合规权限的用户和项目（全部）

此报告列出了每个用户和项目组合，其中用户被分配到不允许向用户授予其当前在项目中拥有的所有权限的 SAG。该报告包括所有用户，无论他们当前在项目中是否已过期。

项目页面

项目状态选项卡

该模块添加了一个页面，显示项目中所有用户的状态。每个用户的状态由用户的 SAG 及其在项目中拥有的权限决定。该行的颜色指示用户是否：

  • 绿色 - 符合 SAG 要求

  • 红色 - 不符合 SAG 要求

  • 灰色——项目中已过期

您还可以检查“不合规权限”列，了解用户拥有哪些 SAG 不允许的权限。

💡
 如果您只想查看不当拥有特定权限的用户，可以使用搜索框。例如，如果您仅对用户权限和/或项目设计和设置权限感兴趣，请键入"user rights" | "project design"搜索框中的"user rights" | "project design" 。有关详细信息，请参阅搜索部分。

  • 当用户以现有用户角色添加到项目时

  • 当用户的权限改变时

  • 当角色的权限定义更改时

  • 当用户添加到用户角色/从用户角色中删除时

  • 当用户通过 CSV 导入到项目中时

  • 通过 CSV 导入更改用户权限时

  • 当通过 CSV 将角色导入项目时

  • 通过 CSV 导入将用户分配给角色时

  • 当通过 API 将用户添加到项目时

  • 当通过 API 更改用户权限时

翻译（国际化/本地化）

一般信息

该模块能够通过外部模块框架的语言选择功能翻译成英语以外的语言。要在系统范围内设置模块的语言，请转至控制中心 > 外部模块 > 管理外部模块，然后单击该模块的“配置”按钮。然后从“语言”下拉列表中选择您要使用的语言，然后单击“保存”。

您可以通过访问项目模块管理器并单击模块的“配置”按钮来覆盖项目级别的系统范围语言。然后从“语言”下拉列表中选择您要使用的语言，然后单击“保存”。

🚩
 该模块的某些部分不是由模块本身翻译的，而是依赖于 REDCap 的内置语言翻译系统。其中包括用户权限的名称和相关描述。

局限性

从右到左的语言

目前对 RTL 语言的支持有限。该模块将正确显示 RTL 语言，但 UI 元素的结构/格式仍为 LTR。有一些选项可以更全面地支持 RTL，但这将是低优先级，除非我们收到需要此功能的团体的消息。

翻译模块

该模块提供的翻译是使用自动翻译软件创建的，可能不准确。如果您想更正翻译或贡献新翻译，请按照以下说明操作。

创建新翻译

如果您想将模块翻译成新语言，可以按照以下步骤操作：

  1. 在模块的源代码中找到lang目录。

  2. 复制English.ini文件并将副本名称更改为您要翻译的语言。使用该语言的英文名称（大写）命名该文件，后跟括号中的该语言的本地名称（使用该语言的字形）。例如，如果您想将模块翻译成日语，您可以将文件命名Japanese (日本語).ini 。

  3. 在文本编辑器中打开刚刚创建的文件，并翻译每行等号右侧的文本。例如，如果您想将文本Introduction翻译成日语，则可以将行status_ui_3 = "Introduction"更改为status_ui_3 = "序章" 。

  4. 对每一行重复该过程。

  5. 保存文件并将其上传到模块源代码的“lang”文件夹中。

  6. 将包含更改的拉取请求提交到Github 存储库的 master 分支。

更正现有翻译

如果您想更正现有翻译，可以按照以下步骤操作：

  1. 在模块的源代码中找到lang目录。

  2. 在文本编辑器中打开要更正的语言的文件。

  3. 更正要更改的每一行等号右侧的文本。

  4. 保存文件并将其上传到模块源代码的“lang”文件夹中。

  5. 将包含更改的拉取请求提交到Github 存储库的 master 分支。

各种各样的

搜寻中

一般的

模块中的许多表格都有一个可用于过滤表格的搜索框。搜索框将搜索表中的所有列。

例如，如果您想要查找当前分配给 SAG 且其标签中包含'Nothing'一词的所有用户，则可以在搜索框中键入'Nothing' ，表将被过滤为仅显示具有“Nothing”字样的行。任何列中的文本'Nothing' 。

图 26. 搜索示例

“AND”搜索

默认情况下，搜索框将是“AND”搜索，这意味着它将您的搜索词拆分为单独的单词，并且仅显示与所有这些搜索词匹配的结果。例如，如果您在搜索框中键入'joe admin' ，则表格将被过滤为仅显示任何列中同时包含'joe'和'admin'行。它不会显示具有'joe'或'admin'的行，但会显示例如第 1 列中具有'admin'和第 2 列中'joe'行。

图 27. AND 示例

“或”搜索

如果要显示与一个值匹配或与另一值匹配的所有行，则必须使用“OR”搜索。

您可以使用“|”运算符以非此即彼的方式搜索多个类似的术语。例如，如果您想查找用户名为"alice"或"bob"的所有用户，您可以在搜索框中输入"alice | bob" ，表格将被过滤为仅显示具有"alice"或"bob"的用户"bob"在任何列中。

💡
 包括“|”符号具有使搜索成为正则表达式搜索的副作用（见下文）。
图 28. 或示例

精确短语搜索

如果您想根据短语进行过滤，可以将短语放在双引号中。例如，如果您想查找包含确切短语'joe admin'的所有行，您可以在搜索框中键入'"joe admin"' ，表格将被过滤为仅显示任何列中包含'joe admin'的行。

图 29. 精确短语示例

正则表达式搜索

通过包含“|”搜索词中任何位置的字符，您的搜索就会变成正则表达式搜索。这使您可以编写默认搜索无法实现的复杂搜索。

图 30.正则表达式示例- 显示"d"后跟任意数量的字母，然后是"n"的所有行 - 如"admin"和"dan"
