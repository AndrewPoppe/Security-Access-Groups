介绍

常问问题

  1. 那是什么？

    这是一个外部模块，允许 REDCap 管理员在 REDCap 系统中创建和管理安全访问组。

  2. 什么是安全访问组？

    安全访问组(SAG) 用于限制 REDCap 用户在项目中可被授予的用户权限。SAG 并不定义用户在特定项目中拥有的具体权限，而是定义用户可以被授予的权限集合。SAG 在系统级别定义，并应用于任何启用了此模块的项目。

  3. 什么时候会用到呢？

    例如，如果您的机构要求用户提供数据导出培训的证明文件，您可以创建一个安全授权组 (SAG)，该授权组不允许向分配到该授权组的用户授予“数据导出工具”用户权限。用户完成培训后，可以将其移至允许授予“数据导出工具”用户权限的安全授权组。

    换句话说，您可以为用户提供不同级别的培训，并仅向完成相应培训的用户授予特定的项目级用户权限。虽然此模块不负责管理或跟踪培训，但它可以处理其他所有相关事宜。

  4. 管理用户权限难道不是研究团队的职责吗？

    是也不是。作为系统管理员，我们有责任确保系统用户接受过充分的培训，能够胜任他们的工作。许多用户如果只是在数据采集工具中输入数据，并不愿意参加长达数小时的培训。您可以为这些用户提供更简短、更有针对性的培训，并将他们分配到权限最低的SAG（服务授权组）。

如何使用该模块？

REDCap 管理员首先通过控制中心的菜单创建服务授权组 (SAG)。创建 SAG 后，可以在同一控制中心菜单的另一个选项卡中将用户分配到该 SAG。用户被分配到某个 SAG 后，只能被授予该 SAG 允许的用户权限。

🚩
必须在项目中启用该模块才能限制该项目中的用户权限。建议在所有项目中启用该模块。

当该模块在项目中启用时，它会在授予用户任何权限之前，根据用户的服务授权协议 (SAG) 检查其允许的权限。该模块会拦截项目中所有设置用户权限的尝试，包括

  • 使用用户权限项目页面上的常用方法

  • 使用“用户权限”项目页面上的“上传用户、角色和分配”功能

  • 使用 REDCap API

如果用户未根据其服务授权协议 (SAG) 获得相应权限，则所有设置用户权限的尝试都将被阻止。任何试图绕过该模块的行为都会被检测并记录。

💡

该模块会忽略项目中当前已过期的用户。这允许用户保留在项目中，但会阻止他们使用不合规的用户权限，直到这些不合规的权限被移除。移除后，该用户即可恢复其过期权限。

该模块提供了一种便捷的方式，可以在“项目状态”选项卡上使项目中的用户过期。

报告选项卡

图 11. 报告选项卡

此选项卡提供了一种简便的方法来查看系统中所有当前拥有与其当前服务授权组 (SAG) 不符的用户权限的用户。这种情况可能发生在首次在项目中启用该模块时，或者当用户被分配到新的 SAG 时。

报告选项如下：

报告标题描述

拥有不合规权限的用户（未过期）

本报告列出了所有被分配到 SAG 的用户，这些 SAG 不允许用户获得其在项目中当前拥有的全部权限。本报告仅包含在项目中权限未过期的用户。

权限不合规的用户（全部）

本报告列出了所有被分配到某个SAG（软件授权组）的用户，该SAG不允许用户获得其在项目中当前拥有的全部权限。本报告包含所有用户，无论其在项目中的权限是否已过期。

具有不合规权利的项目（未过期）

本报告列出了所有至少包含一名用户的项目，该用户被分配到的SAG权限不足，无法授予其在项目中拥有的全部权限。本报告仅包含用户帐户未过期的用户。

存在不合规权利的项目（全部）

本报告列出了所有至少包含一名用户的项目，该用户被分配到的SAG（服务授权组）不允许授予该用户其在项目中当前拥有的所有权限。本报告包含所有用户，无论其用户帐户是否已过期。

权限不合规的用户和项目（未过期）

本报告列出了所有用户与项目的组合，其中用户被分配到的SAG（服务授权组）不允许授予用户其在项目中当前拥有的所有权限。本报告仅包含在项目中权限未过期的用户。

权限不合规的用户和项目（全部）

本报告列出了所有用户和项目组合，其中用户被分配到的 SAG 不允许授予该用户其在项目中当前拥有的所有权限。本报告包含所有用户，无论其在项目中的权限是否已过期。

项目页面

项目状态选项卡

该模块添加了一个页面，用于显示项目中所有用户的状态。每个用户的状态由其服务授权组 (SAG) 和在项目中拥有的权限决定。行的颜色表示该用户的状态：

  • 绿色——符合其SAG标准

  • 红色——不符合其美国演员工会（SAG）的规定。

  • 灰色 - 已在项目中过期

您还可以查看“不合规权限”列，了解用户拥有哪些其 SAG 不允许的权限。

💡
如果您只想查看拥有不当权限的用户，可以使用搜索框。例如，如果您只对“用户权限”和/或“项目设计与设置权限”感兴趣，请在搜索框中输入"user rights" | "project design" 。更多信息，请参阅“搜索”部分。

目前对从右到左 (RTL) 语言的支持有限。该模块可以正确显示 RTL 语言，但 UI 元素的结构/格式仍将沿用从左到右 (LTR) 的格式。我们有一些方案可以更全面地支持 RTL，但除非有用户群体提出此功能需求，否则优先级较低。

本模块提供的译文由自动翻译软件生成，可能不够准确。如果您想更正译文或贡献新的译文，请按照以下说明操作。

创建新翻译

如果您想将模块翻译成另一种语言，请先fork模块GitHub 仓库的main分支。然后，请按照以下步骤操作：

  1. 在模块源代码中找到lang目录。

  2. 复制English.ini文件，并将副本重命名为您想要翻译成的语言。文件名应为该语言的英文名称（首字母大写），后跟该语言的内标（使用该语言的字形），并用括号括起来。例如，如果您想要将模块翻译成日语，则应将文件命名为Japanese (日本語).ini 。

  3. 用文本编辑器打开你刚刚创建的文件，并翻译每一行等号右侧的文本。例如，如果你想把“ Introduction翻译成日语，你需要把status_ui_3 = "Introduction"这行改成status_ui_3 = "序章" 。

  4. 对每一行重复此过程。

  5. 保存文件并将其上传到模块源代码的“lang”文件夹中。

  6. 向Github 仓库的main分支提交包含您的更改的拉取请求。

更正现有翻译

如果要更正现有翻译，可以按照以下步骤操作：

  1. 在模块源代码中找到lang目录。

  2. 用文本编辑器打开要更正的语言文件。

  3. 修改每一行等号右侧的文字。

  4. 保存文件并将其上传到模块源代码的“lang”文件夹中。

  5. 向Github 仓库的main分支提交包含您的更改的拉取请求。

搜索

一般的

模块中的许多表格都带有搜索框，可用于筛选表格内容。搜索框会搜索表格中的所有列。

例如，如果您想查找当前分配给 SAG 且标签中包含'Nothing'一词的所有用户，您可以在搜索框中输入'Nothing' ，表格将被筛选，仅显示任何列中包含文本'Nothing'的行。

图 26. 搜索示例

“AND”搜索

默认情况下，搜索框会使用“与”搜索，这意味着它会将您的搜索词拆分成多个单词，并仅显示与所有单词都匹配的结果。例如，如果您在搜索框中输入'joe admin' ，表格将进行筛选，仅显示在任意列中同时包含'joe'和'admin'行。它不会显示仅包含'joe'或'admin'行，但会显示例如第一列包含'admin'而第二列包含'joe'行。

图 27. 和示例

“或”搜索

如果要显示所有符合某个值或符合另一个值的行，则必须使用“或”搜索。

您可以使用“|”运算符以二选一的方式搜索多个匹配项。例如，如果您想查找所有用户名为"alice"或"bob"的用户，可以在搜索框中输入"alice | bob" ，表格将进行筛选，仅显示任意列中包含"alice"或"bob"用户。

💡
包含“|”符号会产生使搜索成为正则表达式搜索的副作用（见下文）。
图 28. 或示例

精确短语搜索

如果您想根据某个短语进行筛选，可以将该短语用双引号括起来。例如，如果您想查找所有包含短语'joe admin'行，可以在搜索框中输入'"joe admin"' ，表格将被筛选，仅显示任何列中包含'joe admin'行。

图 29. 精确短语示例

正则表达式搜索

在搜索词中任意位置添加“|”字符，即可将搜索转换为正则表达式搜索。这样，您就可以创建默认搜索无法实现的复杂搜索。
