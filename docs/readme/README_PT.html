Introdução

Perguntas frequentes

  1. O que é?

    Este é um Módulo Externo que permite aos administradores do REDCap criar e gerenciar Grupos de Acesso de Segurança em um sistema REDCap.

  2. O que são grupos de acesso de segurança?

    Grupos de acesso de segurança (SAGs) são usados ​​para restringir quais direitos de usuário um usuário REDCap pode receber em um projeto. Os SAGs não definem os direitos que um usuário terá em um determinado projeto; em vez disso, eles definem o conjunto de direitos permitidos que o usuário pode receber. Os SAGs são definidos no nível do sistema e são usados ​​em qualquer projeto que tenha esse módulo habilitado.

  3. Quando isso seria útil?

    Por exemplo, se sua instituição requer documentação de treinamento para usuários exportarem dados, você pode criar um SAG que não permita que o Direito de Usuário "Ferramenta de Exportação de Dados" seja concedido aos usuários atribuídos a ele. Depois que um usuário conclui o treinamento, ele pode ser movido para um SAG que permite que o direito de usuário "Ferramenta de exportação de dados" seja concedido.

    Em outras palavras, você pode querer fornecer diferentes níveis de treinamento para seus usuários e permitir apenas direitos de usuário específicos no nível do projeto para usuários que concluíram o treinamento apropriado. Embora este módulo não administre ou monitore o treinamento, ele lida com todo o resto.

  4. Não é responsabilidade da equipe de estudo gerenciar os direitos do usuário?

    Sim e não. Como administradores de sistema, temos a responsabilidade de garantir que os usuários de nosso sistema sejam adequadamente treinados para os trabalhos que realizarão. Muitos usuários não desejam concluir um treinamento de horas se forem apenas inserir dados em um instrumento de coleta de dados. Você pode dar a esses usuários um treinamento mais curto e focado e atribuí-los a um SAG que permite direitos mínimos de usuário.

Como você usa o módulo?

Os administradores do REDCap primeiro criam SAGs usando um menu no Centro de Controle. Depois que um SAG é criado, os usuários podem ser atribuídos a ele em outra guia do mesmo menu do centro de controle. Quando um usuário é atribuído a um SAG, o usuário só poderá receber direitos de usuário permitidos pelo SAG.

🚩
 O módulo deve ser ativado em um projeto para restringir os direitos do usuário nesse projeto. Recomenda-se que o módulo esteja habilitado em todos os projetos.

Quando o módulo é ativado em um projeto, ele verifica quais direitos são permitidos para um usuário de acordo com seu SAG antes de conceder quaisquer direitos de usuário a esse usuário. O módulo intercepta todas as tentativas de definir direitos de usuário em um projeto, incluindo

  • Usando os métodos usuais na página do projeto User Rights

  • Usando o recurso Carregar usuários, funções e atribuições na página do projeto Direitos do usuário

  • Usando a API REDCap

Todas as tentativas de definir direitos de usuário serão bloqueadas se o usuário não tiver permissão para receber os direitos de acordo com seu SAG. As tentativas de burlar o módulo são detectadas e registradas.

💡

O módulo ignora os usuários que estão atualmente expirados no projeto. Isso permite que o usuário permaneça no projeto, mas o impede de usar seus direitos de usuário não compatível até que esses direitos não compatíveis sejam removidos. O usuário pode então ser un-expirado.

O módulo fornece uma maneira conveniente de expirar usuários em um projeto na guia Status do projeto .

Guia Relatórios

Figura 11. Guia Relatórios

Esta guia fornece uma maneira fácil de ver todos os usuários no sistema que atualmente possuem direitos de usuário que não estão de acordo com seu SAG atual. Isso pode ocorrer quando o módulo é habilitado pela primeira vez em um projeto ou quando um usuário é atribuído a um novo SAG.

As opções de relatório são as seguintes:

Título do relatório Descrição

Usuários com direitos não compatíveis (não expirados)

Este relatório lista todos os usuários atribuídos a um SAG que não permite que o usuário receba todos os direitos que possui atualmente em um projeto. Este relatório inclui apenas usuários se eles não estiverem expirados no(s) projeto(s).

Usuários com direitos não conformes (todos)

Este relatório lista todos os usuários atribuídos a um SAG que não permite que o usuário receba todos os direitos que possui atualmente em um projeto. Este relatório inclui todos os usuários, independentemente de estarem expirados no(s) projeto(s).

Projetos com direitos não conformes (não expirados)

Este relatório lista todos os projetos que possuem pelo menos um usuário atribuído a um SAG que não permite que o usuário receba todos os direitos que possui atualmente no projeto. Este relatório inclui apenas usuários que têm uma conta de usuário não expirada.

Projetos com direitos não conformes (todos)

Este relatório lista todos os projetos que possuem pelo menos um usuário atribuído a um SAG que não permite que o usuário receba todos os direitos que possui atualmente no projeto. Este relatório inclui todos os usuários, independentemente de sua conta de usuário ter expirado.

Usuários e projetos com direitos não conformes (não expirados)

Este relatório lista todas as combinações de usuário e projeto nas quais o usuário é atribuído a um SAG que não permite que o usuário receba todos os direitos que possui atualmente no projeto. Este relatório inclui apenas usuários que não expiraram no projeto.

Usuários e projetos com direitos não conformes (todos)

Este relatório lista todas as combinações de usuário e projeto nas quais o usuário é atribuído a um SAG que não permite que o usuário receba todos os direitos que possui atualmente no projeto. Este relatório inclui todos os usuários, independentemente de estarem expirados no projeto.

Para incluir projetos de Produção e Desenvolvimento, você usaria project_status=Production | project_status=Development . Consulte a seção Pesquisa para obter mais informações.

Página do projeto

Aba Status do Projeto

O módulo adiciona uma página que mostra o status de todos os usuários do projeto. O status de cada usuário é determinado pelo SAG do usuário e pelos direitos que ele possui no projeto. A cor da linha indica se o usuário é:

  • Verde - compatível com seu SAG

  • Vermelho - não compatível com seu SAG

  • Cinza - expirado no projeto

Você também pode verificar a coluna Direitos não compatíveis para ver quais direitos o usuário possui que não são permitidos por seu SAG.

💡
 Se você quiser ver apenas os usuários que possuem direitos específicos de forma inadequada, você pode usar a caixa de pesquisa. Por exemplo, se você estiver interessado apenas nos direitos do usuário e/ou design e configuração do projeto , digite "user rights" | "project design" na caixa de pesquisa. Consulte a seção Pesquisa para obter mais informações.

Você também pode expirar o usuário do projeto. Um alerta pode opcionalmente ser enviado ao usuário e/ou aos detentores dos direitos do usuário quando o usuário expirar.

Tradução (i18n / l10n)

Informações gerais

O módulo pode ser traduzido para outros idiomas além do inglês por meio do recurso de seleção de idioma do External Module Framework. Para definir o idioma do módulo em todo o sistema, vá para Centro de Controle > Módulos Externos > Gerenciar Módulos Externos e clique no botão "Configurar" para o módulo. Em seguida, selecione o idioma que deseja usar no menu suspenso "Idioma" e clique em "Salvar".

Você pode substituir o idioma de todo o sistema no nível do projeto, visitando o Project Module Manager e clicando no botão "Configurar" para o módulo. Em seguida, selecione o idioma que deseja usar no menu suspenso "Idioma" e clique em "Salvar".

🚩
 Há partes do módulo que não são traduzidas pelo próprio módulo, mas, em vez disso, dependem do sistema de tradução de idiomas integrado do REDCap. Isso inclui os nomes dos direitos do usuário e as descrições associadas.

Limitações

Idiomas da direita para a esquerda

Atualmente, há suporte limitado para idiomas RTL. O módulo exibirá os idiomas RTL corretamente, mas a estrutura/formatação dos elementos da interface do usuário ainda será LTR. Existem opções para oferecer suporte mais completo ao RTL, mas isso será de baixa prioridade, a menos que tenhamos notícias de grupos que precisam desse recurso.

Traduzindo o Módulo

As traduções fornecidas com o módulo foram criadas usando um software de tradução automática e podem não ser precisas. Se você gostaria de corrigir uma tradução ou contribuir com uma nova tradução, siga as instruções abaixo.

Criando uma nova tradução

Se você deseja traduzir o módulo para um novo idioma, pode fazê-lo seguindo estas etapas:

  1. Encontre o diretório lang no código-fonte do módulo.

  2. Copie o arquivo English.ini e altere o nome da cópia para o idioma para o qual deseja traduzir. Nomeie o arquivo com o nome em inglês do idioma (em letras maiúsculas) seguido pelo endônimo do idioma (usando os glifos desse idioma) entre parênteses. Por exemplo, se quiser traduzir o módulo para o japonês, nomeie o arquivo Japanese (日本語).ini .

  3. Abra o arquivo que você acabou de criar em um editor de texto e traduza o texto do lado direito do sinal de igual para cada linha. Por exemplo, se você quiser traduzir o texto Introduction para o japonês, altere a linha status_ui_3 = "Introduction" para status_ui_3 = "序章" .

  4. Repita o processo para cada linha.

  5. Salve o arquivo e carregue-o na pasta "lang" do código-fonte do módulo.

  6. Envie uma solicitação pull com suas alterações para a ramificação principal do repositório Github .

Corrigindo uma tradução existente

Se você deseja corrigir uma tradução existente, pode fazê-lo seguindo estas etapas:

  1. Encontre o diretório lang no código-fonte do módulo.

  2. Abra o arquivo para o idioma que deseja corrigir em um editor de texto.

  3. Corrija o texto à direita do sinal de igual para cada linha que deseja alterar.

  4. Salve o arquivo e carregue-o na pasta "lang" do código-fonte do módulo.

  5. Envie uma solicitação pull com suas alterações para a ramificação principal do repositório Github .

Em geral

Muitas das tabelas do módulo possuem uma caixa de pesquisa que pode ser usada para filtrar a tabela. A caixa de pesquisa pesquisará todas as colunas da tabela.

Por exemplo, se você deseja encontrar todos os usuários atualmente atribuídos ao SAG cujo rótulo contém a palavra 'Nothing' , digite 'Nothing' na caixa de pesquisa e a tabela será filtrada para mostrar apenas as linhas que têm o texto 'Nothing' em qualquer coluna.

Figura 26. Exemplo de pesquisa

Pesquisas "E"

Por padrão, a caixa de pesquisa será uma pesquisa 'AND', o que significa que ela dividirá seu termo de pesquisa em palavras separadas e mostrará apenas os resultados que correspondem a todas essas palavras de pesquisa. Por exemplo, se você digitar 'joe admin' na caixa de pesquisa, a tabela será filtrada para mostrar apenas as linhas com 'joe' E 'admin' em qualquer coluna. Ele não mostrará linhas que tenham 'joe' ou 'admin' , mas mostrará linhas que tenham, por exemplo, 'admin' na coluna 1 e 'joe' na coluna 2.

Figura 27. Exemplo de AND

Pesquisas "OR"

Se você quiser mostrar todas as linhas que correspondem a um valor OU correspondem a outro valor, você deve usar uma pesquisa "OR".

Você pode usar o "|" operador para pesquisar vários termos como este de uma maneira ou/ou. Por exemplo, se você deseja encontrar todos os usuários com o nome de usuário "alice" ou "bob" , digite "alice | bob" na caixa de pesquisa e a tabela será filtrada para mostrar apenas os usuários com "alice" ou "bob" em qualquer coluna.

 incluindo o '|' O símbolo tem o efeito colateral de tornar a pesquisa uma pesquisa de expressão regular (veja abaixo ).

Pesquisas de Frases Exatas

Se você deseja filtrar com base em uma frase, pode colocá-la entre aspas duplas. Por exemplo, se você deseja localizar todas as linhas com a frase exata 'joe admin' , digite '"joe admin"' na caixa de pesquisa e a tabela será filtrada para mostrar apenas as linhas com 'joe admin' em qualquer coluna.

Figura 29. Exemplo de frase exata

Pesquisas de expressões regulares

Incluindo um '|' caractere em qualquer lugar do termo de pesquisa, sua pesquisa se transformará em uma pesquisa de expressão regular . Isso permite compor pesquisas complexas que não são possíveis com a pesquisa padrão.
