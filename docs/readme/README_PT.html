Introdução

Perguntas frequentes

  1. O que é?

    Este é um módulo externo que permite aos administradores REDCap criar e gerenciar grupos de acesso de segurança em um sistema REDCap.

  2. O que são grupos de acesso de segurança?

    Grupos de acesso de segurança (SAGs) são usados ​​para restringir quais direitos de usuário um usuário REDCap pode receber em um projeto. Os SAGs não definem os direitos que um usuário terá em um determinado projeto; em vez disso, eles definem o conjunto de direitos permitidos que o usuário pode obter. SAGs são definidos no nível do sistema e são utilizados em qualquer projeto que tenha este módulo habilitado.

  3. Quando isso seria útil?

    Por exemplo, se sua instituição exige documentação de treinamento de usuários para exportação de dados, você pode criar um SAG que não permite que o direito de usuário "Ferramenta de Exportação de Dados" seja concedido aos usuários a ela atribuídos. Depois que um usuário conclui o treinamento, ele pode ser transferido para um SAG que permite a concessão do direito de usuário da "Ferramenta de exportação de dados".

    Em outras palavras, você pode querer fornecer diferentes níveis de treinamento aos seus usuários e permitir apenas direitos de usuário específicos em nível de projeto para usuários que tenham concluído o treinamento apropriado. Embora este módulo não administre ou monitore o treinamento, ele cuida de todo o resto.

  4. Não é responsabilidade da equipe de estudo gerenciar os direitos dos usuários?

    Sim e não. Como administradores de sistema, temos a responsabilidade de garantir que os usuários do nosso sistema sejam adequadamente treinados para os trabalhos que realizarão. Muitos usuários não desejam concluir um treinamento de horas de duração se apenas inserirem dados em um instrumento de coleta de dados. Você pode oferecer a esses usuários um treinamento mais curto e focado e atribuí-los a um SAG que permita direitos mínimos de usuário.

Como você usa o módulo?

Os administradores do REDCap primeiro criam SAGs usando um menu no Centro de Controle. Uma vez criado um SAG, os usuários podem ser atribuídos a ele em outra aba do mesmo menu do centro de controle. Quando um usuário é atribuído a um SAG, o usuário só poderá receber Direitos de Usuário permitidos pelo SAG.

🚩
 O módulo deve estar habilitado em um projeto para restringir os direitos do usuário nesse projeto. Recomenda-se que o módulo esteja habilitado em todos os projetos.

Quando o módulo estiver habilitado em um projeto, ele verificará quais direitos são permitidos para um usuário de acordo com seu SAG antes de conceder quaisquer direitos de usuário a esse usuário. O módulo intercepta todas as tentativas de definir direitos de usuário em um projeto, incluindo

  • Usando os métodos usuais na página do projeto Direitos do usuário

  • Usando o recurso Carregar usuários, funções e atribuições na página do projeto Direitos do usuário

  • Usando a API REDCap

Todas as tentativas de definir direitos de usuário serão bloqueadas se o usuário não puder receber os direitos de acordo com seu SAG. As tentativas de contornar o módulo são detectadas e registradas.

💡

O módulo ignora usuários que estão atualmente expirados no projeto. Isso permite que o usuário permaneça no projeto, mas evita que ele use seus direitos de usuário não conformes até que esses direitos não conformes sejam removidos. O usuário pode então ser cancelado.

O módulo fornece uma maneira conveniente de expirar usuários em um projeto na guia Status do Projeto .

Guia Relatórios

Figura 11. Guia Relatórios

Esta guia fornece uma maneira fácil de ver todos os usuários no sistema que atualmente possuem direitos de usuário que não estão em conformidade com seu SAG atual. Isto pode ocorrer quando o módulo é habilitado pela primeira vez em um projeto ou quando um usuário é atribuído a um novo SAG.

As opções de relatório são as seguintes:

Título do relatório Descrição

Usuários com direitos não conformes (não expirados)

Este relatório lista todos os usuários atribuídos a um SAG que não permite que o usuário receba todos os direitos que possui atualmente em um projeto. Este relatório inclui apenas usuários que não estejam atualmente expirados no(s) projeto(s).

Usuários com direitos não conformes (todos)

Este relatório lista todos os usuários atribuídos a um SAG que não permite que o usuário receba todos os direitos que possui atualmente em um projeto. Este relatório inclui todos os usuários, independentemente de eles estarem atualmente expirados no(s) projeto(s).

Projetos com direitos não conformes (não expirados)

Este relatório lista todos os projetos que possuem pelo menos um usuário atribuído a um SAG que não permite que o usuário receba todos os direitos que possui atualmente no projeto. Este relatório inclui apenas usuários que possuem uma conta de usuário não expirada.

Projetos com direitos não conformes (todos)

Este relatório lista todos os projetos que possuem pelo menos um usuário atribuído a um SAG que não permite que o usuário receba todos os direitos que possui atualmente no projeto. Este relatório inclui todos os usuários, independentemente de sua conta de usuário ter expirado.

Usuários e projetos com direitos não conformes (não expirados)

Este relatório lista cada combinação de usuário e projeto em que o usuário está atribuído a um SAG que não permite que o usuário receba todos os direitos que possui atualmente no projeto. Este relatório inclui apenas usuários que não estão atualmente expirados no projeto.

Usuários e projetos com direitos não conformes (todos)

Este relatório lista cada combinação de usuário e projeto em que o usuário está atribuído a um SAG que não permite que o usuário receba todos os direitos que possui atualmente no projeto. Este relatório inclui todos os usuários, independentemente de estarem atualmente expirados no projeto.

Figura 12. Exemplo de relatório
💡
 Você pode filtrar com base no status do projeto incluindo "project_status=" e o status que deseja filtrar. Por exemplo, para incluir apenas projetos que estão em produção, você usaria project_status=Production . Para incluir apenas projetos concluídos, você usaria project_status=Completed . Para incluir projetos de produção e desenvolvimento, você usaria project_status=Production | project_status=Development . Consulte a seção Pesquisando para obter mais informações.

Página do projeto

Guia Status do Projeto

O módulo adiciona uma página que mostra o status de todos os usuários do projeto. O status de cada usuário é determinado pelo SAG do usuário e pelos direitos que ele possui no projeto. A cor da linha indica se o usuário é:

  • Verde - compatível com seu SAG

  • Vermelho - não compatível com seu SAG

  • Cinza - expirou no projeto

Você também pode verificar a coluna Direitos não conformes para ver quais direitos o usuário possui e que não são permitidos pelo seu SAG.

💡
 Se quiser ver apenas usuários que possuem direitos específicos de forma inadequada, você pode usar a caixa de pesquisa. Por exemplo, se você estiver interessado apenas nos direitos do usuário e/ou direitos de design e configuração do projeto , digite "user rights" | "project design" na caixa de pesquisa. Consulte a seção Pesquisando para obter mais informações.
Figura 14. Aba Status do Projeto
💡
 Se houver algum usuário que não esteja em conformidade com seu SAG, você poderá usar um dos botões de Ação para enviar um e-mail ao usuário, aos detentores dos direitos do usuário ou a ambos. Você também pode expirar o usuário do projeto. Opcionalmente, um alerta pode ser enviado ao usuário e/ou aos titulares dos direitos do usuário quando o usuário expirar.

Tradução (i18n / l10n)

Informações gerais

O módulo pode ser traduzido para outros idiomas além do inglês por meio do recurso de seleção de idioma da Estrutura de Módulo Externo. Para definir o idioma do módulo em todo o sistema, vá para Centro de Controle > Módulos Externos > Gerenciar Módulos Externos e clique no botão "Configurar" do módulo. Em seguida, selecione o idioma que deseja usar no menu suspenso “Idioma” e clique em “Salvar”.

Você pode substituir o idioma de todo o sistema no nível do projeto visitando o Gerenciador de Módulo de Projeto e clicando no botão "Configurar" do módulo. Em seguida, selecione o idioma que deseja usar no menu suspenso “Idioma” e clique em “Salvar”.

🚩
 Existem partes do módulo que não são traduzidas pelo próprio módulo, mas dependem do sistema de tradução de idiomas integrado do REDCap. Estes incluem os nomes dos direitos do usuário e as descrições associadas.

Limitações

Idiomas da direita para a esquerda

Atualmente há suporte limitado para idiomas RTL. O módulo exibirá os idiomas RTL corretamente, mas a estrutura/formatação dos elementos da UI ainda será LTR. Existem opções para oferecer suporte mais completo ao RTL, mas isso terá baixa prioridade, a menos que recebamos notícias de grupos que precisam desse recurso.

Traduzindo o Módulo

As traduções fornecidas com o módulo foram criadas utilizando software de tradução automática e podem não ser precisas. Se você quiser corrigir uma tradução ou contribuir com uma nova tradução, siga as instruções abaixo.

Criando uma nova tradução

Se você deseja traduzir o módulo para um novo idioma, primeiro bifurque a ramificação main do repositório Github do módulo. Em seguida, siga estas etapas:

  1. Encontre o diretório lang no código-fonte do módulo.

  2. Copie o arquivo English.ini e altere o nome da cópia para o idioma para o qual deseja traduzir. Nomeie o arquivo com o nome em inglês do idioma (em maiúscula) seguido do endônimo do idioma (usando os glifos desse idioma) entre parênteses. Por exemplo, se você quiser traduzir o módulo para o japonês, nomeie o arquivo Japanese (日本語).ini .

  3. Abra o arquivo que você acabou de criar em um editor de texto e traduza o texto do lado direito do sinal de igual de cada linha. Por exemplo, se você quisesse traduzir o texto Introduction para o japonês, você alteraria a linha status_ui_3 = "Introduction" para status_ui_3 = "序章" .

  4. Repita o processo para cada linha.

  5. Salve o arquivo e carregue-o na pasta “lang” do código-fonte do módulo.

  6. Envie uma solicitação pull com suas alterações para o branch main do repositório Github .

Corrigindo uma tradução existente

Se quiser corrigir uma tradução existente, você pode fazê-lo seguindo estas etapas:

  1. Encontre o diretório lang no código-fonte do módulo.

  2. Abra o arquivo do idioma que deseja corrigir em um editor de texto.

  3. Corrija o texto do lado direito do sinal de igual para cada linha que deseja alterar.

  4. Salve o arquivo e carregue-o na pasta “lang” do código-fonte do módulo.

  5. Envie uma solicitação pull com suas alterações para o branch main do repositório Github .

Procurando

Em geral

Muitas das tabelas do módulo possuem uma caixa de pesquisa que pode ser usada para filtrar a tabela. A caixa de pesquisa pesquisará todas as colunas da tabela.

Por exemplo, se você deseja encontrar todos os usuários atualmente atribuídos ao SAG cujo rótulo contém a palavra 'Nothing' , você pode digitar 'Nothing' na caixa de pesquisa e a tabela será filtrada para mostrar apenas as linhas que possuem o texto 'Nothing' em qualquer coluna.

Figura 26. Exemplo de pesquisa

Pesquisas "E"

Por padrão, a caixa de pesquisa será uma pesquisa 'AND', o que significa que ela dividirá o termo de pesquisa em palavras separadas e mostrará apenas os resultados que correspondem a todas essas palavras de pesquisa. Por exemplo, se você digitar 'joe admin' na caixa de pesquisa, a tabela será filtrada para mostrar apenas linhas com AMBOS 'joe' E 'admin' em qualquer coluna. Ele não mostrará linhas que possuem 'joe' ou 'admin' mas mostrará linhas que possuem, por exemplo, 'admin' na coluna 1 e 'joe' na coluna 2.

Figura 27. E exemplo

Pesquisas "OU"

Se você quiser mostrar todas as linhas que correspondem a um valor OU a outro valor, você deve usar uma pesquisa "OU".

Você pode usar o "|" operador para pesquisar vários termos como este de uma maneira ou/ou. Por exemplo, se você deseja encontrar todos os usuários com o nome de usuário "alice" ou "bob" , você pode digitar "alice | bob" na caixa de pesquisa e a tabela será filtrada para mostrar apenas os usuários com "alice" ou "bob" em qualquer coluna.

💡
 incluindo o '|' O símbolo tem o efeito colateral de tornar a pesquisa uma pesquisa de expressão regular (veja abaixo ).
Figura 28. Exemplo OR

Pesquisas de frases exatas

Se quiser filtrar com base em uma frase, você pode colocá-la entre aspas duplas. Por exemplo, se quiser encontrar todas as linhas com a frase exata 'joe admin' você pode digitar '"joe admin"' na caixa de pesquisa e a tabela será filtrada para mostrar apenas linhas com 'joe admin' em qualquer coluna.

Figura 29. Exemplo de frase exata

Pesquisas de expressões regulares

Incluindo um '|' qualquer caractere em qualquer lugar do termo de pesquisa, sua pesquisa se transformará em uma pesquisa de expressão regular . Isso permite compor pesquisas complexas que não são possíveis com a pesquisa padrão.

Figura 30. Exemplo de expressão regular - mostrando todas as linhas onde um "d" é seguido por qualquer número de letras e depois um "n" - como em "admin" e "dan"
